Release notes:

Verzió 1.8

Javított hibák:

Basic tokenizáló hibásan ismert fel néhány utasítást (PL: INK-nek ismerte fel az INKEY$ utasítást).

Új funkciók

Külön szín állítható be mostantól a BASIC szerkesztõben a megjegyzésekhez, és függvényekhez.

A BASIC szerkesztõben lehetõség van a BASIC program automatikus újra számozására, ehhez be kell állítanunk a kezdõ sorszámot, és a sornövekményt. A sorszámok az ugró utasításokban automatikusan korrigálásra kerülnek. A következõ utasításokban a sorszám nem kerül korrigálásra:
LIST, LLIST, DELETE, NEW

Lehetõség van a felesleges szóközök eltávolítására, a lefordított BASIC programban. Ebben az esetben a TVC-re készült BASIC kód nem tartalmaz szóközöket, ezáltal a programméret jelentõsen csökkenthetõ. Elõfordulhat azonban, hogy a program mûködésképtelenné válik, mert a TVC rosszul reagál bizonyos esetekben a szóköz hiányára. Ennek elkerülésére használjunk zárójelezést.

 

Verzió 1.7

Javított hibák:

A DW utasítás hibásan kezelte a szimbólumokat. Az utasításban ezentúl mind a konstans, mind pedig a cím szimbólumok egyaránt használhatóak, lehetõvé téve például cím táblázatok létrehozását.

Az emulátor csak akkor indul el fordítás után, ha a CAS file sikeresen létrejött.

Az assembler a string literálokon belül nem végzi el a kis és nagybetûs konverziót.

 

 Verzió 1.6

Javított hibák:

A Lista és Basic loader állományok megnyitáskor azonnal aktívvá válnak.

A kimeneti ablak automatikusan a legvégére gördül, így mindig az utolsó fordítás eredményét mutatja.

A program tartalma néha nem került mentésre, így a régi tartalom fordult.

Ha a WAV file generálása során hiba lépett fel, a program nem írta ki a sikertelenség okát.

Új funkciók:

A program gombok ezentúl nem tûnnek el a program listában, hanem inaktívvá válnak, így nem zavaró, ha a törlés gomb például a futtatás helyére ugrik.

Bekapcsolható opció: A kimeneti ablak tartalma automatikusan törölhetõ fordítás elõtt, így mindig csak az aktuális fordítás eredményét tartalmazza.

Bekapcsolható opció: A program azonnali indítása emulátorban sikeres fordítás után.

A gyorskeresés bekerült a felsõ szerkesztés menübe is.

Új funkció: Keresés és csere a programkódban.

Verzió 1.4

Javított hibák:

Ha két különbözõ útvonalról nyitottunk meg programot ugyan azzal a névvel, akkor csak az elsõ útvonalon levõ program nyílt meg.

Az INCBIN utasításban csak teljes file útvonalat lehetett megadni. Most már a program helyéhez képest relatív útvonalat is használhatunk.

Az INCBIN utasítás nem volt a szerkesztõben kiszínezve.

Az csak kommentet tartalmazó assembly sorok nem kerülnek formázásra.

A DJNZ utasítás bizonyos esetekben hibásan számolta az ugrási címet. ( Pl: Önmagára történõ ugrás nem volt lehetséges)

Az alapértelmezett beállítás használatakor az assembly kód szekciói (komment, utasítás) nem kerültek visszaállításra.

Ha nem volt beállítva az emulátor útvonala, akkor a program „null reference” üzenetet küldött emulátorban történõ programfuttatás esetén.

Ha a DB utasításban csak egy karakteres szöveget használtunk, akkor az nem került be a programba.

A DB utasítás elfogadja a szövegeken belül megadott vesszõ karaktert.

Új funkciók:

A program két indítás között megjegyzi a fõ ablak méretét, és helyét.

Wav file készítésekor további három paramétert is beállíthatunk (fázistolás, bevezetõ block hosszúság, blokkok közötti GAP).

 F12-vel az adott szimbólumot definiáló sorba ugorhatunk (csak assembly programok esetében).

CTRL+Plus,CTRL-Minus gombokkal a szerkesztõben zoomolhatunk.

 CTRL+Balra: Navigálás a szerkesztõben szavanként balra

CTRL+Jobbra: Navigálás az editorban szavanként jobbra.
